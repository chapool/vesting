## 1. 代币基本信息

### 1.1 基础参数

- **代币名称**: HZ Token
- **代币符号**: HZ
- **代币总量**: 10,000,000,000 HZ (100亿枚)
- **小数位数**: 18 (与 USDT、ETH 保持一致)
- **标准协议**: ERC-20 + 扩展功能
- **网络**: Ethereum Mainnet
- **合约地址**: 待部署确定

### 1.2 供应模型

**固定供应量模型**
- 总量固定，无通胀机制
- 通过销毁机制实现通缩
- 保护持有者长期利益

**发行策略**
- 一次性铸造全部代币
- 通过 Vesting 合约控制释放
- 按预定计划分批释放

## 2. 分配方案

### 2.1 总体分配

| 分配类型 | 数量 (HZ) | 占比 | 用途说明 |
|---------|-----------|------|----------|
| 挖矿奖励 | 6,000,000,000 | 60% | 激励生态参与者 |
| 运营与生态 | 2,000,000,000 | 20% | 项目运营和生态建设 |
| 团队和顾问 | 1,000,000,000 | 10% | 团队激励和顾问报酬 |
| 基石轮投资 | 1,000,000,000 | 10% | 早期投资者 |
| **总计** | **10,000,000,000** | **100%** | - |

### 2.2 分配策略详解

#### 2.2.1 挖矿奖励 (60%)

**目标用户**: 生态参与者、矿工、流动性提供者
**分配目的**: 
- 激励长期参与生态建设
- 维护网络安全和稳定
- 促进代币流通和使用

**使用场景**:
- 流动性挖矿奖励
- 质押挖矿收益
- 社区贡献奖励
- 推荐奖励

#### 2.2.2 运营与生态 (20%)

**目标用户**: 合作伙伴、用户、开发者
**分配目的**:
- 支持项目市场推广
- 激励生态合作伙伴
- 资助社区活动

**使用场景**:
- 空投活动
- 合作伙伴激励
- 市场营销活动
- 开发者资助
- 社区建设

#### 2.2.3 团队和顾问 (10%)

**目标用户**: 核心团队、技术顾问、战略顾问
**分配目的**:
- 激励团队长期参与
- 吸引优秀人才加入
- 确保项目持续发展

**锁仓机制**:
- 1年完全锁仓期
- 2年线性释放期
- 与项目发展深度绑定

#### 2.2.4 基石轮投资 (20%)

**目标用户**: 早期投资者、机构投资者
**分配目的**:
- 获得早期资金支持
- 引入战略合作伙伴
- 建立行业影响力

**投资轮次**:
- 第一轮: 10% (机构投资者)
- 第二轮: 10% (战略投资者)

## 3. 释放机制

### 3.1 挖矿奖励释放机制

**释放模式**: 线性释放
**释放周期**: 60个月 (5年)
**开始时间**: 合约部署后立即开始

```
月度释放量: 100,000,000 HZ
年度释放量: 1,200,000,000 HZ
释放公式: 6,000,000,000 ÷ 60 = 100,000,000 HZ/月
```

**释放曲线**:
```
年份   | 累计释放(亿HZ) | 剩余(亿HZ) | 释放进度
-------|---------------|-----------|----------
第1年  | 12.00         | 48.00     | 20.00%
第2年  | 24.00         | 36.00     | 40.00%
第3年  | 36.00         | 24.00     | 60.00%
第4年  | 48.00         | 12.00     | 80.00%
第5年  | 60.00         | 0         | 100.00%
```

### 3.2 运营与生态释放机制

**释放模式**: 分期释放 (前期加速)
**释放周期**: 60个月 (5年)

```
第1年: 每月 100,000,000 HZ × 12个月 = 1,200,000,000 HZ (60%)
第2-5年: 每月 16,666,667 HZ × 48个月 = 800,000,016 HZ (40%)
```

**释放策略**:
- 前期加速释放，支持项目启动
- 后期平稳释放，维持生态发展
- 灵活调整，适应市场需求

### 3.3 团队和顾问释放机制

**释放模式**: Cliff + 线性释放
**总周期**: 36个月 (1年锁仓 + 2年释放)

```
Cliff期: 12个月 (完全锁仓)
释放期: 24个月 (线性释放)
月度释放: 41,666,667 HZ (从第13个月开始)
```

**激励机制**:
- 长期锁仓确保团队承诺
- 与项目里程碑绑定
- 防止早期大量抛售

### 3.4 基石轮释放机制

**释放模式**: 根据投资协议确定
**基本原则**:
- 平衡投资者利益和项目稳定
- 考虑市场环境和发展阶段
- 可能包含不同的锁仓期

**典型方案**:
```
方案A: 6个月锁仓 + 18个月线性释放
方案B: 12个月锁仓 + 12个月线性释放
方案C: 立即释放20% + 12个月线性释放80%
```

## 5. 经济激励

### 5.1 持有激励

**质押奖励**
- 质押 HZ 获得额外收益
- 长期质押获得更高倍率
- 支持流动性质押

**治理权益**
- 持有者参与项目治理
- 投票权重与持有量相关
- 重要决策社区表决

### 5.2 使用激励

**交易挖矿**
- 交易产生手续费返还
- 大额交易享受折扣
- 活跃用户特殊奖励

**生态激励**
- 使用生态产品获得奖励
- 推荐新用户获得代币
- 参与社区活动获得空投

### 5.3 流动性激励

**LP 奖励**
- 提供流动性获得挖矿奖励
- 不同交易对不同倍率
- 根据市场需求动态调整

**做市商激励**
- 专业做市商特殊奖励
- 维护价格稳定性
- 提高市场深度

## 6. 价值捕获

### 6.1 代币价值来源

**使用价值**
- 生态系统内的通用货币
- 支付交易手续费
- 购买生态产品服务

**治理价值**
- 参与项目重要决策
- 影响发展方向
- 享受治理收益

**投资价值**
- 随生态发展增值
- 稀缺性带来溢价
- 机构认可提升价值

### 6.2 价值稳定机制

**供应控制**
- 固定总量防止通胀
- 销毁机制实现通缩
- 释放节奏平滑供应

**需求创造**
- 多场景使用需求
- 质押锁仓减少流通
- 生态发展增加需求

**市场稳定**
- 分层释放避免抛压
- 团队锁仓展示信心
- 机构投资提供支撑

## 经济模型总结

### 设计亮点

1. **平衡分配**: 60%挖矿激励确保生态发展，20%运营资金支持项目推广
2. **分层释放**: 不同类型代币采用不同释放策略，平衡各方利益
3. **长期激励**: 团队代币长期锁仓，确保项目长期发展
4. **灵活调整**: 预留治理机制，支持参数动态优化

### 风险控制

1. **抛售风险**: 通过锁仓和分期释放机制控制
2. **通胀风险**: 固定总量杜绝通胀可能
3. **流动性风险**: 多元化分配确保充足流动性
4. **治理风险**: 去中心化治理机制防止权力集中
